# Example 01: 简单的顺序工作流 - 文章生成器

## 复杂度
⭐ 初级

## 需求

### 背景
实现一个自动化的文章生成系统，通过多个顺序步骤将一个简单的标题转化为完整的、经过润色的文章。

### 功能需求

#### 1. 核心流程
系统需要按照以下顺序执行四个步骤：

**步骤1：标题输入**
- 接收用户提供的文章标题
- 验证标题不为空
- 将标题存入状态

**步骤2：大纲生成**
- 基于标题生成文章大纲
- 大纲应包含3-5个主要章节
- 每个章节有简短的描述

**步骤3：正文撰写**
- 根据大纲展开完整的文章内容
- 每个章节应有2-3段内容
- 总字数约500-800字

**步骤4：内容润色**
- 优化文章的语言表达
- 检查语法和逻辑连贯性
- 提升文章的可读性

#### 2. 状态管理
需要维护以下状态信息：
- `title`: 文章标题
- `outline`: 生成的大纲
- `content`: 正文内容
- `polished_content`: 润色后的最终内容
- `current_step`: 当前执行步骤

#### 3. 输出要求
- 每个步骤完成后显示进度
- 最终输出完整的文章
- 显示整个处理流程的耗时

### 技术要求

#### LangGraph 学习点
- 理解 `StateGraph` 的基本用法
- 掌握节点（Node）的定义和添加
- 理解状态（State）的定义和传递
- 掌握边（Edge）的添加，实现顺序执行
- 学习 `set_entry_point()` 设置入口
- 学习 `compile()` 编译图
- 掌握 `invoke()` 执行图

#### 实现要点
1. 使用 `TypedDict` 定义状态结构
2. 每个节点是一个独立的函数
3. 节点函数接收状态，返回状态更新
4. 使用 `add_edge()` 连接节点
5. 最后一个节点连接到 `END`

### 验收标准

1. **功能完整性**
   - 能够接收标题并生成完整文章
   - 四个步骤按顺序执行
   - 状态正确传递和更新

2. **代码质量**
   - 代码结构清晰，注释完整
   - 状态定义明确
   - 节点职责单一

3. **可运行性**
   - 提供可直接运行的示例代码
   - 包含示例输入
   - 能够看到完整的输出结果

### 示例输入输出

**输入：**
```
标题: "人工智能在医疗领域的应用"
```

**预期输出结构：**
```
=== 第1步：标题确认 ===
标题: 人工智能在医疗领域的应用

=== 第2步：大纲生成 ===
1. 引言：AI医疗的兴起
2. 诊断辅助系统
3. 药物研发加速
4. 个性化治疗方案
5. 未来展望

=== 第3步：正文撰写 ===
[完整的文章内容...]

=== 第4步：内容润色 ===
[润色后的文章内容...]

总耗时: X.XX秒
```

### 扩展思考

完成基础需求后，可以考虑：
1. 添加日志记录每个步骤的执行时间
2. 支持不同的文章风格（正式/轻松/学术）
3. 添加文章质量评分机制
4. 支持中英文双语生成
